## Summary

`ssot/system-overview.md` tree entries include inline numeric counts
(e.g. "29 test cases", "49 tests", "15 tests", "5 SPA routes", "8 sections")
that are manually maintained and go stale with every ticket that adds or
removes tests or routes.

These numbers are not auto-generated by `build-ticket-derived-ssot.mjs` and
have no enforcement gate. The tree's job is to tell readers *what each file
owns*, not how many items it currently has — those counts belong in the spec
runner output, not the doc tree.

This is a docs-only ticket. Zero runtime code changes.

---

## Acceptance Criteria

**AC-070-DROP-COUNTS:** Remove all inline numeric counts from tree entries in
`ssot/system-overview.md`. Targets include (verify with grep at implementation
time — list may be longer):

| Location | Current annotation | Action |
|---|---|---|
| `diary-page.spec.ts` | "29 test cases (DIARY-PAGE-CURATION×9 + ...)" | Drop count; keep tag list if ≤1 line |
| `diary-homepage.spec.ts` | "4 test cases (HOMEPAGE-CURATION 0/1/2/3-entry + tie-break)" | Drop count; keep purpose |
| `about-layout.spec.ts` | "15 tests: container width / ..." | Drop count; keep topic list if ≤1 line |
| `favicon-assets.spec.ts` | "8 asset paths + link-tag hrefs + manifest schema" | Keep content description; drop "8" |
| `ga-spa-pageview.spec.ts` | "9 tests: SPA-NAV × 2 + BEACON × 4 + NEG × 3" | Drop count; keep tag list if ≤1 line |
| `test_predictor.py` | "49 tests, 含 K-013 contract parametrize 3 cases + fixture coverage + realism rule" | Drop "49 tests"; keep the K-013 attribution |
| Summary section | "5 SPA routes" / "8 sections" | Keep — these are architectural facts, not test counts; verify whether they require a Sacred note |

Replacement pattern: `← <file-purpose> (<ticket-attribution>)` — no numeric prefix.

**AC-070-GREP-VERIFY:** After edits, run:
```
grep -n "[0-9]\+ test\|[0-9]\+ tests\|[0-9]\+ cases\|[0-9]\+ asset" ssot/system-overview.md
```
Output must be empty (zero matches).
